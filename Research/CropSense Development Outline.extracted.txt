Stage 2  |  AI Model Development &amp; Training
Week 3 – 5

Objective: Train, evaluate, and benchmark the CNN classification model. Select the best architecture for dual deployment (mobile + embedded). Export to TensorFlow Lite.

Task
Owner
Details
Timeline
Baseline model training (MobileNetV2)
Data Scientist
Implement transfer learning from ImageNet pretrained MobileNetV2; fine-tune on crop dataset; log accuracy, loss, confusion matrix
Day 1–5
Benchmark model training (EfficientNet-Lite)
Data Scientist
Train EfficientNet-Lite0 and Lite2 variants; compare against MobileNetV2 on accuracy, inference speed, and model size
Day 5–10
Model evaluation &amp; selection
Data Scientist
Evaluate all models on test set; generate per-class precision/recall/F1; select final model based on accuracy vs. latency trade-off
Day 10–12
TensorFlow Lite export &amp; quantisation
Data Scientist + Software Developer
Convert selected model to .tflite format; apply INT8 post-training quantisation; verify accuracy loss is within 1%; test inference on test images
Day 12–15
Treatment recommendation database
Software Developer
Create JSON lookup table mapping each of the 22 disease/pest classes to a concise treatment recommendation in English and Twi
Day 8–12

Stage Deliverables:

Trained model achieving &gt;= 92% test accuracy on 22-class dataset
Benchmark comparison table committed to repository
Final .tflite model file (quantised) committed to repository
Treatment recommendation JSON database (English + Twi)

Owner
Details
Timeline

TFLite model integration into app
Software Developer + Data Scientist
Bundle .tflite model in .NET MAUI raw assets; integrate TensorFlow.Lite NuGet package; write image pre-processing pipeline matching training parameters using SkiaSharp
Day 5–10
Camera capture &amp; image input flow
Software Developer
Implement live camera capture and gallery upload; add image cropping/resizing to 224x224 before inference
Day 6–10
Result screen &amp; recommendation display
Software Developer
Display detected disease name, confidence score, crop type, and treatment recommendation; show healthy/warning/critical colour badge
Day 10–14
Offline functionality &amp; app testing
Software Developer
Verify all features work without internet; test on Android and iOS devices of varying specs; fix crashes and UI issues; measure inference time
Day 14–18

Working .NET MAUI app running on Android and iOS

Disease result screen with treatment recommendation in English and Twi
Fully offline — no internet dependency confirmed

Week 5 – 9

Objective: Design and build the CropSense handheld embedded scouting device. 3D-print the enclosure, wire the hardware, and deploy the TFLite model on Raspberry Pi with a Python inference pipeline.

Task
Owner
Details
Timeline
Device enclosure CAD design
CAD / Prototyping

Day 1–8
3D printing &amp; enclosure assembly
CAD / Prototyping
Print enclosure in PLA/PETG; test fit of all components; iterate and reprint if needed; sand and finish for presentation
Day 8–14
Hardware wiring &amp; circuit assembly
Electronics Engineer
Wire Raspberry Pi Zero 2W to OV5647 camera, 2-inch LCD display, status LEDs, push button, and Li-Po battery with protection circuit
Day 1–8
Power management &amp; battery circuit
Electronics Engineer
Implement TP4056-based Li-Po charge circuit; add power switch; test 6-hour runtime target; add low-battery LED indicator
Day 6–12
Raspberry Pi OS setup &amp; TFLite runtime
Software Developer

Day 3–8
Python inference firmware
Software Developer + Electronics Engineer
Write main.py: button-triggered capture, image pre-processing, TFLite inference, result display on LCD, LED colour output (green=healthy, red=diseased), audio beep
Day 8–15
Field simulation testing
All team members
Simulate outdoor use: test under natural light, different backgrounds, partial leaf coverage; measure inference time on device; fix edge cases
Day 15–18

Stage Deliverables:

Printed and assembled device enclosure
Fully wired and powered hardware prototype
Python firmware deployed and running on Raspberry Pi
On-device inference confirmed under 3 seconds
CAD files (.STL and Fusion 360) committed to repository
